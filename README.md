# CS 230 – The Gaming Room Project

## Reflection

The client for this project was The Gaming Room, a company that wanted to expand their existing game, Draw It or Lose It, which was originally only available on Android. Their goal was to redesign the application so it could run as a web-based system and be accessible across multiple platforms such as Windows, Mac, Linux, and mobile devices. The software needed to support multiple teams and players, maintain unique identifiers, and handle real-time gameplay while being scalable and efficient.

One thing I did particularly well in developing this documentation was clearly connecting the technical decisions to the client’s needs. Instead of just describing concepts, I made sure to explain why certain choices like using Linux, a relational database, and a client-server architecture made sense for this specific application.

Working through the design document before coding was very helpful because it forced me to think through the structure of the application ahead of time. It helped me understand how different components would interact, which made the coding process more straightforward.

If I could revise one part of my work, I would improve the system architecture section by adding a visual diagram to better show how components communicate. When interpreting the user’s needs, I focused on scalability, usability, and performance. Considering user needs is important because the success of the software depends on how well it meets those expectations. When designing the software, I used object-oriented principles and broke the system into smaller components. In the future, I would continue using these strategies along with more visual planning tools like UML diagrams.
